# 换换 (HuanHuan) 电商平台 — 项目计划书

## 一、项目概述

换换 (HuanHuan) 是一个创新型电商平台，核心特色是"以物换物"交易模式，结合平台虚拟货币"换呗"(HuanBei) 实现灵活的商品交换与交易。平台支持商户发布闲置/商品，用户可通过换呗购买或发起以物换物请求，平台收取服务费盈利。

项目覆盖 **Web 网页端、Android 客户端、iOS 客户端、微信小程序** 四大终端，共享统一后端 API 服务。

### 核心业务特色

- **以物换物**：用户可发起换物请求，双方协商后完成物品交换
- **换呗货币**：平台虚拟货币，用于商品购买、换物差价补偿
- **商户体系**：所有用户均为商户角色，可同时买卖
- **服务费机制**：平台对每笔交易收取 5% 服务费
- **保证金制度**：商户缴纳保证金保障交易安全

### 5.2 技术栈明细

| 层级 | 技术选型 | 用途 |
|------|---------|------|
| **后端框架** | Node.js + Express + TypeScript | API 服务，业务逻辑 |
| **数据库** | PostgreSQL + Prisma ORM | 数据存储，类型安全的数据访问 |
| **缓存** | Redis | 会话管理、验证码、热数据缓存 |
| **认证** | JWT + bcrypt | 用户认证、密码加密 |
| **参数校验** | Zod | 请求参数类型安全校验 |
| **Web 前端** | React 18 + TypeScript + Ant Design 5 | 网页端 SPA 应用 |
| **状态管理** | Zustand | 轻量级全局状态管理 |
| **路由** | React Router v6 | 前端路由管理 |
| **HTTP 客户端** | Axios | API 请求，拦截器处理 |
| **Android** | Kotlin + Jetpack Compose | 原生 Android 开发 |
| **iOS** | Swift + SwiftUI | 原生 iOS 开发 |
| **微信小程序** | Taro + React + TypeScript | 跨端小程序框架 |
| **构建工具** | Vite | 快速开发构建 |
| **测试** | Vitest + fast-check | 单元测试 + 属性测试 |
| **消息推送** | 极光推送 / APNs / FCM | 交易通知、换物请求提醒 |
